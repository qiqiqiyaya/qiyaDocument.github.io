Claude Code 功能概览

快捷命令 功能说明
Shift + Tab 在计划模式、默认模式和自动编辑模式这三种权限模式之间循环切换 ¹ 。
! (感叹号) + 命令 进入 bash 模式，直接在电脑上运行终端命令，而不是作为对话发给 Claude Code ² 。
# + 输入 直接说你想让 claude 记住什么
Ctrl + B 当终端被运行的项目阻塞时，将该进程放到后台运行，以便继续对话 ² ³ 。
Option + Enter (Mac)
Ctrl + Enter (Win)		 在输入框中进行文本换行（直接按 Enter 会发送消息） ⁴ 。
Ctrl + V 将图片粘贴到对话框中（注意在 Mac 上也是使用 Ctrl+V） ⁴ 。
/help 查看所有可用的斜杠命令以及它们背后的思想 ⁵ 。
/model 切换不同档位（高、中、低）的大模型 ⁵ 。
/btw + 问题 询问与当前项目无关的问题，回答会与原对话的上下文隔离，按 ESC 退出后不占用上下文 ⁵ 。
/simplify 这是一个内置 skill，会派生三个 agent 从代码质量、运行效率和是否可用三个角度做代码审核与自动优化 ⁵ ⁶ 。
连接两下 ESC 或
/rewind		 调出撤回/回滚界面，可以快捷撤销 Claude Code 刚刚编辑过的文件和对话 ⁶ ⁷ 。
/compact 主动压缩上下文，保留关键信息并释放大部分上下文空间，省 token 且让模型更专注 ⁸ 。
/clear 彻底清空当前上下文，相当于重新开启一个新的对话 ⁸ 。
/cost 详细展示当前上下文的占比，包括各个 MCP 和 skill 占用的 token 数量 ⁸ 。
连接两次 Ctrl + C 退出 Claude Code 的对话 ⁹ 。
/resume 调出历史记录，选择并恢复到之前的某一次对话 ⁹ 。
claude - 启动时在命令后加空格和横杠，代表 Continue，直接继续上一次的对话 ⁹ 。
/init 初始化整个项目，自动扫描项目架构并将信息写入项目级的 claude.md 文件中 ¹⁰ 。
/memory 打开记忆选项，可以选择编辑全局或项目级的 claude.md ¹¹ ，也可以用来开启自动记忆 (Auto memory) 功能 ¹² 。
/技能名称 手动调用特定的 skill（例如输入 /frontend 调用前端优化技能） ¹³ 。
/agent 进入引导流程，手动创建子 agent (subagent) 去执行并行任务 ¹⁴ 。
/plugin 进入插件管理页面，可以发现新插件或管理已安装的插件 ¹⁵ ¹⁶ 。
